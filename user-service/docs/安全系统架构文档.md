## 核心组件实现（续）

### 组件4：CustomUserDetailsService（用户加载服务）

#### 作用

Spring Security的"用户信息提供者"：
- 根据用户名/邮箱加载用户（登录时）
- 根据用户ID加载用户（JWT验证时）
- 返回UserDetails对象供Security使用

#### 为什么需要这个类？

**问题**：Spring Security不知道你的用户信息存在哪里

```
Spring Security：我需要验证用户"john_doe"的密码
你的应用：   用户信息在MySQL数据库的users表
Spring Security：我不知道怎么查MySQL！
你的应用：   那我实现UserDetailsService接口告诉你
```

#### 完整代码解析

```java
@Service                    // Spring服务组件
@RequiredArgsConstructor    // Lombok生成构造器
public class CustomUserDetailsService implements UserDetailsService {
    // UserDetailsService：Spring Security的标准接口
    // 只有一个方法：loadUserByUsername()

    // 依赖注入：用户数据访问对象
    private final UserRepository userRepository;

    /**
     * 根据用户名或邮箱加载用户（登录时调用）
     *
     * 什么时候调用？
     * - 用户提交登录表单
     * - Spring Security需要验证密码
     *
     * 执行流程：
     * 1. 从数据库查询用户
     * 2. 找不到→抛异常（登录失败）
     * 3. 找到→转换成UserPrincipal返回
     * 4. Spring Security自动验证密码
     *
     * @param usernameOrEmail 用户名或邮箱
     * @return UserDetails对象
     * @throws UsernameNotFoundException 用户不存在
     */
    @Override
    @Transactional  // 事务注解：确保数据库操作原子性
    public UserDetails loadUserByUsername(@NonNull String usernameOrEmail)
            throws UsernameNotFoundException {
        // @NonNull：Nullability注解，IDE会检查null值

        // 从数据库查询用户
        // findByUsernameOrEmail：自定义查询方法
        // 相当于SQL：SELECT * FROM users
        //           WHERE username = ? OR email = ?
        User user = userRepository.findByUsernameOrEmail(
                usernameOrEmail,  // 第一个参数：username
                usernameOrEmail   // 第二个参数：email（相同值）
            )
            .orElseThrow(() -> new UsernameNotFoundException(
                "User not found with username or email: " + usernameOrEmail
            ));
            // Optional.orElseThrow()：
            // - 找到用户→返回User对象
            // - 没找到→抛出UsernameNotFoundException

        // 转换成Spring Security的UserDetails
        return UserPrincipal.create(user);
        // UserPrincipal：我们自己实现的UserDetails
    }

    /**
     * 根据用户ID加载用户（JWT验证时调用）
     *
     * 什么时候调用？
     * - JwtAuthenticationFilter从JWT解析出用户ID
     * - 需要加载完整的用户信息
     *
     * 为什么需要这个方法？
     * - UserDetailsService接口只有loadByUsername
     * - 我们的JWT存储的是用户ID，不是用户名
     * - 所以额外提供这个方法
     *
     * @param id 用户ID
     * @return UserDetails对象
     * @throws ResourceNotFoundException 用户不存在
     */
    @Transactional
    public UserDetails loadUserById(Long id) {
        User user = userRepository.findById(id)
            .orElseThrow(() -> new ResourceNotFoundException(
                "User",   // 资源类型
                "id",     // 字段名
                id        // 字段值
            ));

        return UserPrincipal.create(user);
    }
}
```

#### UserRepository接口

```java
public interface UserRepository extends JpaRepository<User, Long> {
    // JpaRepository：Spring Data JPA提供的基础接口
    // 自动提供：save(), findById(), findAll(), delete()等方法

    /**
     * 根据用户名或邮箱查询用户
     *
     * 方法命名规则：
     * findBy + 字段名 + Or + 字段名
     *
     * Spring Data JPA自动生成SQL：
     * SELECT * FROM users WHERE username = ? OR email = ?
     */
    Optional<User> findByUsernameOrEmail(String username, String email);

    /**
     * 检查用户名是否存在
     *
     * 返回值：boolean（存在返回true）
     *
     * 自动生成SQL：
     * SELECT COUNT(*) > 0 FROM users WHERE username = ?
     */
    boolean existsByUsername(String username);

    /**
     * 检查邮箱是否存在
     */
    boolean existsByEmail(String email);
}
```

### 组件5：UserPrincipal（用户主体）

#### 作用

实现Spring Security的`UserDetails`接口，封装用户信息供Security使用。

#### 为什么需要这个类？

**问题**：Spring Security不认识你的User实体类

```
Spring Security：我需要检查用户是否被锁定
你的User类：  我有status字段（ACTIVE/SUSPENDED/DELETED）
Spring Security：我只认识isAccountNonLocked()方法！
你的应用：   那我创建UserPrincipal实现UserDetails接口
```

#### 完整代码解析

```java
@Getter                     // Lombok生成getter方法
@AllArgsConstructor         // Lombok生成全参构造器
public class UserPrincipal implements UserDetails {
    // UserDetails：Spring Security的用户详情接口
    // 定义了一系列检查用户状态的方法

    // 用户基本信息
    private Long id;              // 用户ID（不是UserDetails要求的）
    private String username;      // 用户名
    private String email;         // 邮箱（不是UserDetails要求的）
    private String password;      // 密码（BCrypt哈希值）
    private User.UserRole role;   // 角色（STUDENT/INSTRUCTOR/ADMIN）
    private User.UserStatus status;  // 状态（ACTIVE/INACTIVE/SUSPENDED/DELETED）

    /**
     * 静态工厂方法：从User实体创建UserPrincipal
     *
     * 为什么用静态方法而不是构造器？
     * - 语义更清晰：UserPrincipal.create(user)
     * - 可以添加额外的逻辑处理
     * - 符合设计模式
     *
     * @param user User实体对象
     * @return UserPrincipal对象
     */
    public static UserPrincipal create(User user) {
        return new UserPrincipal(
            user.getId(),
            user.getUsername(),
            user.getEmail(),
            user.getPassword(),  // 已经是BCrypt哈希，不需要再加密
            user.getRole(),
            user.getStatus()
        );
    }

    /**
     * 获取用户权限列表
     *
     * Spring Security的权限格式：ROLE_XXX
     * 例如：ROLE_STUDENT, ROLE_ADMIN
     *
     * 为什么要ROLE_前缀？
     * - Spring Security的约定
     * - hasRole("ADMIN") 实际检查 ROLE_ADMIN权限
     * - hasAuthority("ROLE_ADMIN") 才是完整的权限名
     *
     * @return 权限集合
     */
    @Override
    public Collection<? extends GrantedAuthority> getAuthorities() {
        // SimpleGrantedAuthority：Spring Security的简单权限实现
        return List.of(
            new SimpleGrantedAuthority("ROLE_" + role.name())
        );
        // 例如：role=STUDENT → ROLE_STUDENT
    }

    /**
     * 获取密码（BCrypt哈希值）
     *
     * Spring Security会用这个密码验证用户输入
     *
     * @return BCrypt哈希后的密码
     */
    @Override
    public String getPassword() {
        return password;
    }

    /**
     * 获取用户名
     *
     * @return 用户名
     */
    @Override
    public String getUsername() {
        return username;
    }

    /**
     * 账户是否未过期
     *
     * 我们的业务逻辑：
     * - DELETED状态视为过期
     * - 其他状态视为未过期
     *
     * @return true=未过期，false=已过期
     */
    @Override
    public boolean isAccountNonExpired() {
        return status != User.UserStatus.DELETED;
        // DELETED用户无法登录
    }

    /**
     * 账户是否未锁定
     *
     * 我们的业务逻辑：
     * - SUSPENDED（暂停）状态视为锁定
     * - 其他状态视为未锁定
     *
     * 使用场景：
     * - 违规用户被管理员暂停
     * - 多次登录失败自动锁定
     *
     * @return true=未锁定，false=已锁定
     */
    @Override
    public boolean isAccountNonLocked() {
        return status != User.UserStatus.SUSPENDED;
        // SUSPENDED用户无法登录
    }

    /**
     * 凭证（密码）是否未过期
     *
     * 我们的业务：密码不过期
     *
     * 如果需要密码过期功能：
     * - 在User表添加passwordExpireDate字段
     * - return LocalDateTime.now().isBefore(passwordExpireDate);
     *
     * @return true=未过期
     */
    @Override
    public boolean isCredentialsNonExpired() {
        return true;  // 我们不实现密码过期
    }

    /**
     * 账户是否启用
     *
     * 我们的业务逻辑：
     * - 只有ACTIVE状态的用户可以登录
     * - INACTIVE：未激活（注册后未验证邮箱）
     * - SUSPENDED：被暂停
     * - DELETED：已删除
     *
     * @return true=启用，false=禁用
     */
    @Override
    public boolean isEnabled() {
        return status == User.UserStatus.ACTIVE;
    }
}
```

#### 状态检查流程

```
用户尝试登录
    ↓
loadUserByUsername() → 返回UserPrincipal
    ↓
Spring Security自动检查：
    ├─ isEnabled()            → false → DisabledException
    ├─ isAccountNonLocked()   → false → LockedException
    ├─ isAccountNonExpired()  → false → AccountExpiredException
    └─ isCredentialsNonExpired() → false → CredentialsExpiredException
    ↓
验证密码：passwordEncoder.matches()
    ↓
创建认证对象
```

### 组件6：UserService（用户业务服务）

#### 作用

处理所有用户相关的业务逻辑：
- 用户注册
- 用户登录
- 令牌刷新
- 用户登出
- 更新资料

#### 完整代码解析（注册功能）

```java
@Service                    // Spring服务组件
@AllArgsConstructor         // Lombok生成全参构造器
public class UserService {

    // 依赖注入（全部通过构造器，无@Autowired）
    private final UserRepository userRepository;
    private final UserProfileRepository userProfileRepository;
    private final RefreshTokenRepository refreshTokenRepository;
    private final PasswordEncoder passwordEncoder;
    private final AuthenticationManager authenticationManager;
    private final JwtTokenProvider tokenProvider;
    private final RedisTemplate<String, Object> redisTemplate;

    /**
     * 用户注册
     *
     * 业务流程：
     * 1. 验证用户名和邮箱唯一性
     * 2. 加密密码
     * 3. 保存用户信息
     * 4. 创建用户资料
     * 5. 生成JWT令牌
     * 6. 保存刷新令牌
     * 7. 缓存用户信息到Redis
     * 8. 返回认证响应
     *
     * @param request 注册请求DTO
     * @return 包含令牌的认证响应
     * @throws BadRequestException 用户名或邮箱已存在
     */
    @Transactional  // 事务：全部成功或全部回滚
    public AuthResponse register(RegisterRequest request) {

        // === 步骤1：验证唯一性 ===

        // 检查用户名是否已存在
        if (userRepository.existsByUsername(request.getUsername())) {
            throw new BadRequestException("Username is already taken");
            // 为什么抛异常而不是返回false？
            // - 异常会自动触发事务回滚
            // - 异常会被全局异常处理器捕获，返回统一格式的错误
        }

        // 检查邮箱是否已存在
        if (userRepository.existsByEmail(request.getEmail())) {
            throw new BadRequestException("Email is already in use");
        }

        // === 步骤2：创建用户对象 ===

        User user = User.builder()  // 建造者模式（Lombok @Builder）
            .username(request.getUsername())
            .email(request.getEmail())

            // ⭐ 重点：密码加密
            .password(passwordEncoder.encode(request.getPassword()))
            // encode()做了什么？
            // 1. 生成随机盐（Salt）
            // 2. 盐 + 密码 → BCrypt哈希
            // 3. 返回：$2a$10${22字符盐}{31字符哈希}
            // 例如：
            // 输入：123456
            // 输出：$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy

            .role(User.UserRole.valueOf(request.getRole()))
            // valueOf()：字符串转枚举
            // 例如："STUDENT" → UserRole.STUDENT

            .status(User.UserStatus.ACTIVE)
            // 新注册用户默认激活
            // 如果需要邮箱验证，应该设为INACTIVE

            .build();

        // === 步骤3：保存用户到数据库 ===

        user = userRepository.save(user);
        // save()做了什么？
        // 1. INSERT INTO users (...) VALUES (...)
        // 2. 数据库生成ID
        // 3. 返回带ID的User对象

        // 为什么要重新赋值？
        // 因为保存后user.getId()才有值

        // === 步骤4：创建用户资料 ===

        UserProfile profile = UserProfile.builder()
            .user(user)  // 外键关联
            .firstName(request.getFirstName())
            .lastName(request.getLastName())
            .build();

        userProfileRepository.save(profile);
        // 为什么分两个表？
        // - users表：核心认证信息（用户名、密码、角色）
        // - user_profiles表：扩展信息（姓名、头像、简介等）
        // - 分离关注点，提高查询性能

        // === 步骤5：生成认证令牌 ===

        // 创建认证对象（用于生成JWT）
        UsernamePasswordAuthenticationToken authToken =
            new UsernamePasswordAuthenticationToken(
                request.getUsername(),
                request.getPassword()  // 原始密码，不是哈希
            );

        // 执行认证（验证密码）
        Authentication authentication =
            authenticationManager.authenticate(authToken);
        // authenticate()做了什么？
        // 1. 调用CustomUserDetailsService.loadUserByUsername()
        // 2. 获取数据库中的密码哈希
        // 3. passwordEncoder.matches(输入密码, 数据库哈希)
        // 4. 成功→返回认证对象，失败→抛异常

        // 为什么注册后还要认证？
        // - 确保密码正确保存
        // - 生成标准的认证对象
        // - 复用登录逻辑

        // === 步骤6：生成JWT令牌 ===

        String accessToken = tokenProvider.generateAccessToken(authentication);
        // 访问令牌：1小时有效期，包含用户信息

        String refreshToken = tokenProvider.generateRefreshToken(user.getId());
        // 刷新令牌：7天有效期，只包含用户ID

        // === 步骤7：保存刷新令牌到数据库 ===

        RefreshToken refreshTokenEntity = RefreshToken.builder()
            .token(refreshToken)
            .user(user)

            // 计算过期时间
            .expiryDate(LocalDateTime.now().plusSeconds(
                tokenProvider.getRefreshExpirationMs() / 1000
            ))
            // 例如：当前时间 + 7天

            .build();

        refreshTokenRepository.save(refreshTokenEntity);
        // 为什么要保存刷新令牌？
        // - 可以撤销（用户登出时删除）
        // - 可以限制数量（一个用户最多N个刷新令牌）
        // - 可以记录使用情况

        // === 步骤8：缓存用户信息到Redis ===

        cacheUserInfo(user.getId(), accessToken);
        // 缓存内容：用户ID、令牌、时间戳
        // 缓存时间：1小时（与访问令牌同步）
        // 作用：
        // - 加快JWT验证速度
        // - 实现令牌黑名单（删除缓存=令牌失效）

        // === 步骤9：构造响应 ===

        return AuthResponse.builder()
            .accessToken(accessToken)
            .refreshToken(refreshToken)
            .expiresIn(tokenProvider.getJwtExpirationMs())
            .user(UserResponse.fromUser(user, profile))
            .build();
        // 返回给前端：
        // {
        //   "accessToken": "eyJhbGci...",
        //   "refreshToken": "eyJhbGci...",
        //   "expiresIn": 3600000,
        //   "user": {
        //     "id": 1,
        //     "username": "john_doe",
        //     "email": "john@example.com",
        //     "role": "STUDENT"
        //   }
        // }
    }

    /**
     * 缓存用户信息到Redis
     *
     * @param userId 用户ID
     * @param accessToken 访问令牌
     */
    private void cacheUserInfo(Long userId, String accessToken) {
        String cacheKey = "user:session:" + userId;
        // Redis Key命名规范：类型:功能:标识

        Map<String, Object> sessionData = Map.of(
            "userId", userId,
            "accessToken", accessToken,
            "timestamp", System.currentTimeMillis()
        );

        redisTemplate.opsForValue().set(
            cacheKey,
            sessionData,
            tokenProvider.getJwtExpirationMs(),  // 过期时间
            TimeUnit.MILLISECONDS
        );
        // Redis会在1小时后自动删除这个Key
    }
}
```

## 完整业务流程

### 流程1：用户注册

#### 前端请求

```javascript
// 前端代码（React）
const handleRegister = async () => {
  const response = await fetch('http://localhost:8080/api/auth/register', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      username: 'john_doe',
      email: 'john@example.com',
      password: 'SecurePass123',
      firstName: 'John',
      lastName: 'Doe',
      role: 'STUDENT'
    })
  });

  const data = await response.json();
  // 保存令牌到LocalStorage
  localStorage.setItem('accessToken', data.accessToken);
  localStorage.setItem('refreshToken', data.refreshToken);
};
```

#### 后端处理流程

```
1. HTTP POST /api/auth/register
   ↓
2. Spring Security过滤链
   ├─ JwtAuthenticationFilter（跳过，无令牌）
   ├─ AuthorizationFilter（检查URL，/api/auth/**允许匿名）
   └─ 通过！
   ↓
3. AuthController.register()
   ↓
4. @Valid验证RegisterRequest
   ├─ username长度3-20 ✅
   ├─ email格式正确 ✅
   ├─ password符合规则 ✅
   └─ 验证通过
   ↓
5. UserService.register()
   ├─ 检查用户名唯一性
   ├─ 检查邮箱唯一性
   ├─ BCrypt加密密码
   ├─ 保存用户
   ├─ 创建用户资料
   ├─ 生成JWT令牌
   ├─ 保存刷新令牌
   └─ 缓存到Redis
   ↓
6. 返回200 OK
   {
     "accessToken": "...",
     "refreshToken": "...",
     "expiresIn": 3600000,
     "user": {...}
   }
```
